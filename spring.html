단골 면접 질문 & 모범 답변
Q.01 Spring IoC / DI 란 무엇인가요? ▼
"IoC(제어의 역전)는 객체의 생성과 생명주기 관리를 개발자가 아닌 Spring 컨테이너가 담당하는 것입니다.
DI(의존성 주입)는 IoC를 구현하는 방법으로, 컨테이너가 필요한 의존 객체를 외부에서 주입해줍니다.
생성자 주입을 권장하는 이유는 불변성 보장, 테스트 용이성, 순환 의존성 컴파일 타임 감지 때문입니다."
Q.02 @Controller vs @RestController 차이는? ▼
"@Controller는 View 이름을 반환해 ViewResolver를 통해 HTML 페이지를 렌더링합니다.
@RestController는 @Controller + @ResponseBody의 합성 어노테이션으로,
반환 값을 JSON/XML로 직렬화하여 HTTP 응답 바디에 직접 씁니다. REST API 개발에 사용합니다."
Q.03 Bean의 스코프(Scope) 종류는? ▼
"기본값인 Singleton은 컨테이너당 인스턴스 하나를 공유합니다.
Prototype은 요청할 때마다 새 인스턴스를 생성합니다.
웹 스코프로는 Request(HTTP 요청마다), Session(세션마다), Application(서블릿 컨텍스트마다)이 있습니다.
Singleton Bean에 Prototype Bean을 주입할 때는 Proxy 처리가 필요합니다."
Q.04 @Transactional 동작 원리는? ▼
"Spring AOP 프록시를 통해 동작합니다. 메서드 호출 전 트랜잭션을 시작(BEGIN)하고,
정상 완료 시 COMMIT, RuntimeException 발생 시 ROLLBACK합니다.
같은 클래스 내부에서 @Transactional 메서드를 self-invoke하면 프록시를 거치지 않아 트랜잭션이 적용되지 않습니다."
Q.05 Filter vs Interceptor vs AOP 차이는? ▼
"Filter는 서블릿 컨테이너 레벨에서 동작해 Spring 영역 밖(인코딩, CORS, 인증 전처리)에 사용합니다.
Interceptor는 DispatcherServlet과 Controller 사이에서 동작하며 HandlerMethod 정보를 활용할 수 있습니다.
AOP는 메서드 레벨에서 동작해 비즈니스 로직 전후(로깅, 트랜잭션)에 사용합니다."
Q.06 JPA N+1 문제란? ▼
"1번의 쿼리로 N개의 엔티티를 조회한 후, 연관 엔티티를 로딩하기 위해 N번의 추가 쿼리가 발생하는 문제입니다.
예: 게시글 10개 조회(1) → 각 게시글의 작성자 조회(10) = 총 11번 쿼리
해결 방법: Fetch Join (JPQL), @EntityGraph, Batch Size 설정으로 해결합니다."
